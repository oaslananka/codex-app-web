# ADR 0002: Protocol Artifact Sync

## Status

Accepted

## Context

codex-app-web depends on OpenAI Codex app-server protocol artifacts generated by
the Codex CLI. Those artifacts change independently of this UI and are security
sensitive because they define approval, terminal, filesystem, auth, plugin, MCP,
and configuration surfaces.

## Decision

Protocol artifacts are generated from a pinned upstream Codex release and
tracked in the repository with metadata. Changes must run manifest and drift
checks so protocol-surface additions, removals, and breaking categories are
visible in review.

## Consequences

- `codex-official-docs/` and
  `src/lib/codex-runtime/official-manifest.generated.ts` are generated inputs,
  not hand-edited protocol definitions.
- `pnpm protocol:manifest:check` verifies generated manifest freshness.
- `pnpm protocol:drift:check` compares protocol surface against the base ref.
- Breaking drift requires a companion runtime, UI, test, or automation change.
