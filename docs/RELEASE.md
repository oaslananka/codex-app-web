# Release Process

## Release Target

The first production release target is GitHub Release from
`oaslananka/codex-app-web`. npm, GHCR, package registries, and static
hosting are not release targets for this backend-enabled app unless a future
repository change adds that surface intentionally.

## Version Source

Versions and tags are derived only from release-please manifest mode:

- `release-please-config.json`
- `.release-please-manifest.json`
- Conventional Commit history

Do not provide manual version, tag, or changelog inputs. Do not create release
tags locally.

## Workflow

1. Merge a reviewed PR into `main` in the canonical GitHub repository.
2. `.github/workflows/release.yml` runs release-please.
3. If release-please only opens or updates a release PR, artifact publication is
   skipped.
4. If release-please creates a GitHub Release, CI checks out clean source,
   installs with Corepack and pnpm 11, validates the repository, builds the app,
   and creates release assets.
5. The workflow uploads the package tarball, CycloneDX SBOM, and SHA256 checksum
   file to the GitHub Release.
6. GitHub artifact attestations are generated for the release assets.
7. The workflow verifies the GitHub Release asset set after upload.

The canonical GitHub repository must allow GitHub Actions to create pull requests
under **Settings > Actions > General > Workflow permissions** so release-please
can open or update the release PR. The release workflow does not approve pull
requests, auto-merge, or publish registry packages.

## Required Assets

- `codex-app-web-<version>.tgz`
- `codex-app-web-<version>.sbom.cdx.json`
- `SHA256SUMS.txt`
- GitHub artifact attestation records
- Release notes generated by release-please

## Local Checks

Before merging release-affecting work, run:

```bash
corepack enable
corepack prepare pnpm@11.3.0 --activate
pnpm install --frozen-lockfile
pnpm format:check
pnpm lint
pnpm repo:hygiene:check
pnpm license:scan
pnpm protocol:manifest:check
pnpm protocol:drift:check
pnpm docs:check
pnpm typecheck
pnpm test
pnpm build
pnpm security:bootstrap
pnpm security:scan
pnpm release:state
```

`pnpm release:state` must report GitHub Release as the configured guarded target.
